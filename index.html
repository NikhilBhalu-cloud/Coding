Introduction to C#

Learn the fundamentals of C# programming language

Theory

C# (pronounced "C-Sharp") is a modern, object-oriented programming language developed by Microsoft. It runs on the .NET framework and is designed for building Windows applications, web applications, and more.

Key Features:
  • Type Safety: Strong type checking prevents runtime errors
  • Memory Management: Automatic garbage collection
  • Object-Oriented: Supports inheritance, polymorphism, and encapsulation
  • Cross-Platform: Runs on Windows, macOS, and Linux with .NET Core

Questions & Answers

C# is a general-purpose programming language used for developing desktop applications, web applications, mobile apps, games, and web services. It's particularly popular for Windows development and enterprise applications.

C# is the programming language, while .NET is the framework/platform that provides libraries, runtime, and tools to execute C# applications. Think of C# as the language and .NET as the ecosystem.

Example Code

using System;

namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            // This is a simple "Hello World" program
            Console.WriteLine("Hello, World!");
            
            // Variables and basic operations
            string name = "C# Developer";
            int year = 2024;
            
            Console.WriteLine($"Welcome {name} to C# programming in {year}!");
            
            // Wait for user input before closing
            Console.ReadLine();
        }
    }
}

Data Types & Variables

Understanding C# data types and variable declarations

Theory

C# is a strongly-typed language where every variable must have a declared type. Data types specify the type of data that variables can store.

Value Types:
  • int - 32-bit integer (-2,147,483,648 to 2,147,483,647)
  • double - 64-bit floating point number
  • bool - Boolean (true/false)
  • char - Single Unicode character
  • decimal - 128-bit decimal for financial calculations
Reference Types:
  • string - Sequence of characters
  • object - Base type for all types
  • arrays - Collection of elements

Example Code

using System;

class DataTypesExample
{
    static void Main()
    {
        // Value types
        int age = 25;
        double salary = 50000.50;
        bool isEmployed = true;
        char grade = 'A';
        decimal price = 99.99m;
        
        // Reference types
        string name = "John Doe";
        object obj = "This can be anything";
        
        // Type inference with var
        var automaticInt = 100; // compiler infers int
        var automaticString = "Hello"; // compiler infers string
        
        // Display values
        Console.WriteLine($"Name: {name}");
        Console.WriteLine($"Age: {age}");
        Console.WriteLine($"Salary: {salary:C}"); // Currency format
        Console.WriteLine($"Employed: {isEmployed}");
        Console.WriteLine($"Grade: {grade}");
        Console.WriteLine($"Price: {price}");
        
        // Checking types
        Console.WriteLine($"Type of automaticInt: {automaticInt.GetType()}");
        Console.WriteLine($"Type of automaticString: {automaticString.GetType()}");
    }
}

Classes & Objects

Understanding object-oriented programming fundamentals

Theory

A class is a blueprint or template that defines the structure and behavior of objects. An object is an instance of a class that contains actual data.

Key Concepts:
  • Fields: Variables that store data
  • Properties: Controlled access to fields
  • Methods: Functions that define behavior
  • Constructors: Special methods for object initialization

Questions & Answers

Fields are variables that store data directly, while Properties provide controlled access to fields through get/set accessors. Properties can include validation, computation, or logging.

Example Code

using System;

public class Car
{
    // Fields (private data)
    private string make;
    private string model;
    private int year;
    private double mileage;
    
    // Properties (controlled access)
    public string Make 
    { 
        get { return make; }
        set { make = value?.Trim(); } // Validation/processing
    }
    
    public string Model 
    { 
        get { return model; }
        set { model = value?.Trim(); }
    }
    
    public int Year 
    { 
        get { return year; }
        set 
        { 
            if (value >= 1886 && value <= DateTime.Now.Year + 1)
                year = value;
            else
                throw new ArgumentException("Invalid year");
        }
    }
    
    // Auto-implemented property
    public string Color { get; set; }
    
    // Read-only property
    public double Mileage => mileage;
    
    // Constructors
    public Car() // Default constructor
    {
        make = "Unknown";
        model = "Unknown";
        year = DateTime.Now.Year;
        mileage = 0;
    }
    
    public Car(string make, string model, int year) // Parameterized constructor
    {
        Make = make;
        Model = model;
        Year = year;
        mileage = 0;
    }
    
    // Methods
    public void Drive(double miles)
    {
        if (miles > 0)
        {
            mileage += miles;
            Console.WriteLine($"Drove {miles} miles. Total: {mileage}");
        }
    }
    
    public void DisplayInfo()
    {
        Console.WriteLine($"{Year} {Make} {Model} - {mileage:F1} miles");
    }
    
    // Override ToString for string representation
    public override string ToString()
    {
        return $"{Year} {Make} {Model}";
    }
}

// Usage example
class Program
{
    static void Main()
    {
        // Create objects
        Car car1 = new Car(); // Using default constructor
        Car car2 = new Car("Toyota", "Camry", 2022); // Using parameterized constructor
        
        // Set properties
        car1.Make = "Honda";
        car1.Model = "Civic";
        car1.Year = 2023;
        car1.Color = "Blue";
        
        car2.Color = "Red";
        
        // Use methods
        car1.Drive(150.5);
        car2.Drive(75.0);
        
        // Display information
        car1.DisplayInfo();
        car2.DisplayInfo();
        
        Console.WriteLine($"Car 1: {car1}"); // Uses ToString()
        Console.WriteLine($"Car 2: {car2}");
    }
}

Operators in C#

Learn about different types of operators and their usage

Theory

Operators are symbols that perform specific operations on one or more operands. C# provides a rich set of operators for arithmetic, comparison, logical, assignment, and other operations.

Types of Operators:
  • Arithmetic Operators: +, -, *, /, % (modulo)
  • Comparison Operators: ==, !=, <, >, <=, >=
  • Logical Operators: && (AND), || (OR), ! (NOT)
  • Assignment Operators: =, +=, -=, *=, /=, %=
  • Increment/Decrement: ++, --
  • Bitwise Operators: &, |, ^, ~, <<, >>

Example Code

using System;

class OperatorsExample
{
    static void Main()
    {
        // Arithmetic operators
        int a = 10, b = 3;
        Console.WriteLine($"Addition: {a} + {b} = {a + b}");
        Console.WriteLine($"Subtraction: {a} - {b} = {a - b}");
        Console.WriteLine($"Multiplication: {a} * {b} = {a * b}");
        Console.WriteLine($"Division: {a} / {b} = {a / b}");
        Console.WriteLine($"Modulo: {a} % {b} = {a % b}");
        
        // Comparison operators
        Console.WriteLine($"{a} == {b}: {a == b}");
        Console.WriteLine($"{a} > {b}: {a > b}");
        Console.WriteLine($"{a} <= {b}: {a <= b}");
        
        // Logical operators
        bool x = true, y = false;
        Console.WriteLine($"x && y: {x && y}");
        Console.WriteLine($"x || y: {x || y}");
        Console.WriteLine($"!x: {!x}");
        
        // Assignment operators
        int c = 5;
        c += 3; // c = c + 3
        Console.WriteLine($"After c += 3: {c}");
        
        // Increment/Decrement
        int d = 10;
        Console.WriteLine($"d++: {d++}"); // Post-increment
        Console.WriteLine($"++d: {++d}"); // Pre-increment
    }
}

Control Flow

Master conditional statements and loops to control program execution

Theory

Control Flow statements determine the order in which code is executed. They allow you to make decisions, repeat operations, and branch program execution based on conditions.

Control Statements:
  • if/else: Conditional execution based on boolean expressions
  • switch: Multi-way branching based on value comparison
  • for loop: Iterate a specific number of times
  • while loop: Repeat while condition is true
  • foreach: Iterate through collections
  • break/continue: Control loop execution

Example Code

using System;

class ControlFlowExample
{
    static void Main()
    {
        // if-else statement
        int score = 85;
        if (score >= 90)
            Console.WriteLine("Grade: A");
        else if (score >= 80)
            Console.WriteLine("Grade: B");
        else if (score >= 70)
            Console.WriteLine("Grade: C");
        else
            Console.WriteLine("Grade: F");
        
        // switch statement
        string day = "Monday";
        switch (day)
        {
            case "Monday":
                Console.WriteLine("Start of work week");
                break;
            case "Friday":
                Console.WriteLine("TGIF!");
                break;
            case "Saturday":
            case "Sunday":
                Console.WriteLine("Weekend!");
                break;
            default:
                Console.WriteLine("Regular weekday");
                break;
        }
        
        // for loop
        Console.WriteLine("Counting 1 to 5:");
        for (int i = 1; i <= 5; i++)
        {
            Console.WriteLine($"Count: {i}");
        }
        
        // while loop
        int count = 0;
        while (count < 3)
        {
            Console.WriteLine($"While loop iteration: {count}");
            count++;
        }
        
        // foreach loop
        string[] names = {"Alice", "Bob", "Charlie"};
        foreach (string name in names)
        {
            Console.WriteLine($"Hello, {name}!");
        }
    }
}

Methods & Parameters

Learn to create reusable code blocks with methods and parameters

Theory

Methods are reusable blocks of code that perform specific tasks. They help organize code, reduce duplication, and make programs more modular and maintainable.

Method Components:
  • Access Modifier: public, private, protected, internal
  • Return Type: The type of value returned (void if no return)
  • Method Name: Descriptive name following naming conventions
  • Parameters: Input values passed to the method
  • Method Body: Code that executes when method is called

Example Code

using System;

class MethodsExample
{
    // Method with no parameters and no return value
    static void SayHello()
    {
        Console.WriteLine("Hello, World!");
    }
    
    // Method with parameters and return value
    static int Add(int a, int b)
    {
        return a + b;
    }
    
    // Method with default parameters
    static void Greet(string name, string greeting = "Hello")
    {
        Console.WriteLine($"{greeting}, {name}!");
    }
    
    // Method with params keyword (variable arguments)
    static int Sum(params int[] numbers)
    {
        int total = 0;
        foreach (int num in numbers)
        {
            total += num;
        }
        return total;
    }
    
    // Method with out parameter
    static bool TryParse(string input, out int result)
    {
        return int.TryParse(input, out result);
    }
    
    // Method with ref parameter
    static void Double(ref int number)
    {
        number *= 2;
    }
    
    // Method overloading - same name, different parameters
    static void Print(string message)
    {
        Console.WriteLine($"String: {message}");
    }
    
    static void Print(int number)
    {
        Console.WriteLine($"Number: {number}");
    }
    
    static void Main()
    {
        // Calling methods
        SayHello();
        
        int result = Add(5, 3);
        Console.WriteLine($"5 + 3 = {result}");
        
        Greet("Alice");
        Greet("Bob", "Hi");
        
        int sum = Sum(1, 2, 3, 4, 5);
        Console.WriteLine($"Sum: {sum}");
        
        // Using out parameter
        if (TryParse("123", out int parsed))
        {
            Console.WriteLine($"Parsed: {parsed}");
        }
        
        // Using ref parameter
        int value = 10;
        Double(ref value);
        Console.WriteLine($"Doubled: {value}");
        
        // Method overloading
        Print("Hello");
        Print(42);
    }
}

Arrays & Strings

Work with collections of data and text manipulation

Theory

Arrays store multiple values of the same type in a single variable. Strings are sequences of characters and have many built-in methods for manipulation.

Array Features:
  • Fixed Size: Array size is determined at creation
  • Zero-based Indexing: First element is at index 0
  • Type Safety: All elements must be of the same type
  • Reference Type: Arrays are reference types

Example Code

using System;

class ArraysStringsExample
{
    static void Main()
    {
        // Array declaration and initialization
        int[] numbers = {1, 2, 3, 4, 5};
        string[] names = new string[3] {"Alice", "Bob", "Charlie"};
        
        // Accessing array elements
        Console.WriteLine($"First number: {numbers[0]}");
        Console.WriteLine($"Array length: {numbers.Length}");
        
        // Iterating through arrays
        Console.WriteLine("Numbers:");
        for (int i = 0; i < numbers.Length; i++)
        {
            Console.WriteLine($"numbers[{i}] = {numbers[i]}");
        }
        
        Console.WriteLine("Names:");
        foreach (string name in names)
        {
            Console.WriteLine($"Name: {name}");
        }
        
        // Multidimensional arrays
        int[,] matrix = {{1, 2}, {3, 4}, {5, 6}};
        Console.WriteLine($"Matrix[1,0]: {matrix[1, 0]}");
        
        // Jagged arrays
        int[][] jaggedArray = new int[3][];
        jaggedArray[0] = new int[2] {1, 2};
        jaggedArray[1] = new int[3] {3, 4, 5};
        jaggedArray[2] = new int[1] {6};
        
        // String manipulation
        string text = "Hello, World!";
        Console.WriteLine($"Original: {text}");
        Console.WriteLine($"Length: {text.Length}");
        Console.WriteLine($"Uppercase: {text.ToUpper()}");
        Console.WriteLine($"Substring: {text.Substring(0, 5)}");
        Console.WriteLine($"Contains 'World': {text.Contains("World")}");
        Console.WriteLine($"Replace: {text.Replace("World", "C#")}");
        
        // String splitting and joining
        string sentence = "apple,banana,orange";
        string[] fruits = sentence.Split(',');
        Console.WriteLine("Fruits:");
        foreach (string fruit in fruits)
        {
            Console.WriteLine($"- {fruit}");
        }
        
        string joined = string.Join(" | ", fruits);
        Console.WriteLine($"Joined: {joined}");
        
        // String interpolation
        string name = "Alice";
        int age = 25;
        Console.WriteLine($"Name: {name}, Age: {age}");
    }
}

Constructors & Destructors

Learn about object initialization and cleanup

Theory

Constructors are special methods called when objects are created to initialize their state. Destructors (finalizers) are called when objects are destroyed by the garbage collector.

Constructor Types:
  • Default Constructor: No parameters, provides default initialization
  • Parameterized Constructor: Takes parameters for custom initialization
  • Copy Constructor: Creates object from another object of same type
  • Static Constructor: Initializes static members of the class

Example Code

using System;

public class Person
{
    private string name;
    private int age;
    private static int totalPersons = 0;
    
    // Static constructor - called once when class is first used
    static Person()
    {
        Console.WriteLine("Static constructor called");
        totalPersons = 0;
    }
    
    // Default constructor
    public Person()
    {
        Console.WriteLine("Default constructor called");
        name = "Unknown";
        age = 0;
        totalPersons++;
    }
    
    // Parameterized constructor
    public Person(string name, int age)
    {
        Console.WriteLine($"Parameterized constructor called for {name}");
        this.name = name;
        this.age = age;
        totalPersons++;
    }
    
    // Constructor chaining using 'this'
    public Person(string name) : this(name, 0)
    {
        Console.WriteLine("Constructor with name only called");
    }
    
    // Copy constructor
    public Person(Person other)
    {
        Console.WriteLine("Copy constructor called");
        this.name = other.name;
        this.age = other.age;
        totalPersons++;
    }
    
    // Properties
    public string Name 
    { 
        get { return name; } 
        set { name = value; } 
    }
    
    public int Age 
    { 
        get { return age; } 
        set { age = value; } 
    }
    
    public static int TotalPersons 
    { 
        get { return totalPersons; } 
    }
    
    // Method
    public void DisplayInfo()
    {
        Console.WriteLine($"Person: {name}, Age: {age}");
    }
    
    // Destructor (Finalizer)
    ~Person()
    {
        Console.WriteLine($"Destructor called for {name}");
        totalPersons--;
    }
}

class ConstructorsExample
{
    static void Main()
    {
        Console.WriteLine("Creating persons...");
        
        // Using different constructors
        Person person1 = new Person(); // Default constructor
        person1.DisplayInfo();
        
        Person person2 = new Person("Alice", 25); // Parameterized constructor
        person2.DisplayInfo();
        
        Person person3 = new Person("Bob"); // Constructor chaining
        person3.DisplayInfo();
        
        Person person4 = new Person(person2); // Copy constructor
        person4.DisplayInfo();
        
        Console.WriteLine($"Total persons created: {Person.TotalPersons}");
        
        // Force garbage collection to see destructors
        person1 = null;
        person2 = null;
        person3 = null;
        person4 = null;
        
        GC.Collect();
        GC.WaitForPendingFinalizers();
        
        Console.WriteLine($"Total persons after GC: {Person.TotalPersons}");
    }
}

Beginner Interview Questions

Essential C# questions for entry-level positions

Class:

  • Reference type stored on the heap
  • Supports inheritance
  • Can be null
  • Default constructor always exists

Struct:

  • Value type stored on the stack
  • No inheritance (except from System.ValueType)
  • Cannot be null (unless nullable)
  • Must initialize all fields
// Class example
public class PersonClass
{
    public string Name { get; set; }
    public int Age { get; set; }
}

// Struct example
public struct PersonStruct
{
    public string Name { get; set; }
    public int Age { get; set; }
    
    public PersonStruct(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

Boxing: Converting a value type to object type (heap allocation)

Unboxing: Converting object type back to value type

// Boxing - value type to reference type
int value = 123;
object boxed = value; // Boxing occurs

// Unboxing - reference type to value type  
int unboxed = (int)boxed; // Explicit cast required

// Generic collections avoid boxing
List<int> numbers = new List<int>(); // No boxing
numbers.Add(123); // Direct value type storage

String:

  • Immutable - creates new object for each modification
  • Good for few string operations
  • Performance penalty with many concatenations

StringBuilder:

  • Mutable - modifies internal buffer
  • Better performance for multiple modifications
  • Uses dynamic buffer allocation
// String concatenation (inefficient for loops)
string result = "";
for (int i = 0; i < 1000; i++)
{
    result += i.ToString(); // Creates new string each time
}

// StringBuilder (efficient for multiple operations)
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++)
{
    sb.Append(i.ToString()); // Modifies internal buffer
}
string result2 = sb.ToString();

Intermediate Interview Questions

Advanced concepts for experienced developers

Abstract Class:

  • Can have implementation and abstract methods
  • Single inheritance only
  • Can have constructors, fields, properties
  • Access modifiers allowed

Interface:

  • Only method signatures (until C# 8.0)
  • Multiple inheritance supported
  • No constructors or fields
  • All members implicitly public
// Abstract class
public abstract class Animal
{
    protected string name;
    
    public Animal(string name) 
    { 
        this.name = name; 
    }
    
    public abstract void MakeSound(); // Must implement
    
    public virtual void Sleep() // Can override
    { 
        Console.WriteLine($"{name} is sleeping"); 
    }
}

// Interface
public interface IFlyable
{
    void Fly();
    int MaxAltitude { get; set; }
}

// Implementation
public class Bird : Animal, IFlyable
{
    public Bird(string name) : base(name) { }
    
    public override void MakeSound() 
    { 
        Console.WriteLine("Tweet tweet"); 
    }
    
    public void Fly() 
    { 
        Console.WriteLine($"{name} is flying"); 
    }
    
    public int MaxAltitude { get; set; }
}

Delegate: Type-safe function pointer that can hold references to methods

Event: Special kind of delegate that provides additional restrictions

// Delegate declaration
public delegate void NotificationHandler(string message);

public class Publisher
{
    // Event based on delegate
    public event NotificationHandler OnNotification;
    
    // Regular delegate
    public NotificationHandler DirectDelegate;
    
    public void SendNotification(string message)
    {
        // Trigger event (safe check)
        OnNotification?.Invoke(message);
    }
}

public class Subscriber
{
    public void Subscribe(Publisher pub)
    {
        // Subscribe to event
        pub.OnNotification += HandleNotification;
        
        // Can also use delegate directly
        pub.DirectDelegate += HandleNotification;
    }
    
    private void HandleNotification(string message)
    {
        Console.WriteLine($"Received: {message}");
    }
}

Advanced Interview Questions

Expert-level concepts and architecture questions

The .NET Garbage Collector (GC) automatically manages memory by:

  1. Generational Collection: Objects are divided into 3 generations
  2. Mark and Sweep: Marks reachable objects, sweeps unreachable ones
  3. Compaction: Moves objects to reduce fragmentation
Generations:
  • Gen 0: Short-lived objects (collected frequently)
  • Gen 1: Medium-lived objects (buffer between 0 and 2)
  • Gen 2: Long-lived objects (collected less frequently)
// Force garbage collection (not recommended in production)
GC.Collect(); // Collect all generations
GC.Collect(0); // Collect only generation 0

// Check memory usage
long memory = GC.GetTotalMemory(false);
Console.WriteLine($"Memory used: {memory} bytes");

// Implement IDisposable for unmanaged resources
public class ResourceHandler : IDisposable
{
    private bool disposed = false;
    
    public void Dispose()
    {
        Dispose(true);
        GC.SuppressFinalize(this);
    }
    
    protected virtual void Dispose(bool disposing)
    {
        if (!disposed)
        {
            if (disposing)
            {
                // Dispose managed resources
            }
            // Dispose unmanaged resources
            disposed = true;
        }
    }
}
